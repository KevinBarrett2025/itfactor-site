Archive is for keeping a project without keeping it in your way.

Archiving moves a project out of Active Projects and into Archives while keeping it available for records, reference, or later restoration. It is the safer choice when you are done with a project but do not want to erase it.

If you are not completely sure, archive instead of delete. That is the simplest rule to keep in mind.

Quick Start

The simplest archive rule for most users.

Archive when

  • You already submitted the deliverable
  • You are done reviewing the project
  • You want to reduce clutter in Active Projects
  • You still want to keep the project record

Delete only when

  • You are sure you do not need the project anymore
  • You are comfortable removing sessions and takes permanently
  • You do not need the project as reference later

The two normal archive entry points

You can archive from the Home-screen project card if the Archive pill is available, or from inside Project Lobby if you are already working in the project.

Part 1

Archive versus delete

These two actions are not interchangeable, and the difference matters.

Archive means
  • The project leaves Active Projects
  • It moves into Archives
  • You can inspect or restore it later
Delete means
  • The project is permanently removed
  • Sessions and takes are removed too
  • It is not the normal cleanup path
Best practical advice
  • If you are unsure, archive first
  • Use delete only when you are certain

Most people want archiving far more often than deletion. It keeps your active dashboard lighter without forcing you to give up the record of the work.

Part 2

How to archive from Home or Project Lobby

The app gives you two normal entry points so you do not have to hunt for the archive action.

From Home
  • Use the Archive pill on a project card when it is available
  • This is the fast path after a project is submitted
From Project Lobby
  • Open the project controls
  • Choose Archive Project
  • This is useful when you decide to archive while already inside the project
The archive screen shows
  • Project title
  • Session and asset counts
  • Whether the project is active or already archived

Choose the entry point that matches where you already are. Home is quicker after submission. Project Lobby is natural if you are still wrapping up inside the project.

Part 3

The main archive choice: keep source videos or not

The most important archive decision is whether the archive keeps the original source videos.

If source videos stay on
  • The archive keeps source videos
  • Deliverables stay
  • Metadata stays
  • This is the fullest archive record
If source videos stay off
  • The archive can trim non-deliverable source video files
  • Deliverables and metadata still stay
  • This helps reduce storage use
Backup bundle options
  • Export Videos
  • Export Metadata
  • These are optional extra local copies, not the same thing as the in-app archive itself

If you want the fullest record, keep source videos in the archive. If you mainly want the project record and final outputs while saving space, trimming source videos can make sense.

Part 4

Where archived projects go and how restore works

Archived projects move into the Archives screen, where you can inspect them one by one or restore them when needed.

Archives shows
  • Total archive storage
  • Project, session, and asset counts
  • Archived project rows with storage and deliverable details
Single-project restore
  • Open the archived project
  • Use the restore action
  • The project returns to Active Projects
Faster restore options
  • Restore from the row in the Archives list
  • Use Restore All if you intentionally want every archived project back

The archive is meant to feel like a calmer storage shelf, not a dead end. Projects stay available there until you decide otherwise.

Part 5

Cleanup and the safest everyday pattern

Archive cleanup exists for when storage starts building up, but it is not something most people need every day.

Use cleanup when
  • Storage has actually built up
  • You want to manage older archived material
The everyday pattern for most users
  • Finish the project
  • Mark the deliverable as submitted if needed
  • Archive the project
  • Come back later only if you need the archive
Why this works
  • It keeps Active Projects cleaner
  • It preserves the project record
  • It avoids permanent decisions too early

Archiving works best when it quietly takes finished work out of your daily workspace without making you feel like you are losing it.

Quick Help

A few archive details explained.

Archive pill on Home

This is the fast path for archiving a finished project from the active dashboard.

Keep Source Videos in Archive

This decides whether the in-app archive keeps the original source videos or trims them to save space.

Backup bundle choices

These create extra local copies in Documents. They are optional and separate from the in-app archive itself.

Project Archive view

When you open an archived project, you are viewing it in archive mode, not as active in-progress work.

Restore All

This brings every archived project back to Active Projects. Use it only when that is truly what you want.

Delete Forever

This is the permanent removal path. It is not the same thing as archiving.

FAQ

Quick answers before you move on.

When should I archive a project?
Usually after the deliverable is sent and you are done actively working in that project.
Should I archive or delete?
Archive is the safer choice if there is any chance you will want the project later.
What does keeping source videos do?
It keeps the fullest version of the archived project, including original source videos, instead of trimming them.
Can I restore an archived project?
Yes. You can restore one archived project or restore them all from the Archives area.
